# M16 — "PLAY MODE" (2D / 3D-clean / 3D-arms)

> Milestone plan. When shipped, retire this file: fold the durable "why" into CLAUDE.md
> and delete M16.md, exactly as M14/M15 were retired. PLAN.md carries no milestone history.

## Context

Today the world-settings board has a single boolean, **"TERRY MOVES PIECES"**
(`PlayerData.TerryMovesPieces` → `SeatedHandSpikes.HandsOn`), toggling whether seated
avatars animate their arms to move pieces. M16 turns that one boolean into a **three-way
"PLAY MODE"** picker and adds a genuinely new rendering path:

- **2D** — every chess board this client sees (tables + north spectator wall) renders as a
  flat, top-down **glyph** board: white/brown squares, black-outlined white/black-filled
  piece sprites (the standard default-2D-chess look) instead of 3D lathed pieces. When you
  sit, the camera pivots to look **straight down** centred on the board. Seated hands **and**
  bodies are off.
- **3D** (`3d-clean`) — today's 3D pieces, seated hands **off**.
- **3D + ARMS** (`3d-arms`) — today's 3D pieces, seated hands **on**. This is the current
  default behaviour, so `3d-arms` is the default and nothing changes for existing players.

Play mode is a **per-client, purely local display choice** (like the old terry-arms toggle,
voice range, and TTS) — each player picks independently, nothing is networked. Confirmed
scope: in 2D, **all boards + the spectator wall** go flat for that client (tables you aren't
seated at will be edge-on since flat glyphs lie on the surface — accepted).

Two hard directions from the owner: standard 2D chess look (white/brown board, outlined
filled pieces), and **"the shader is not the play for input"** — do **not** render the
interactive board with a baked shader-slab; render pieces as flat **mesh quads** reusing the
existing per-piece GameObject machinery so slides/trays/captures/highlights keep working.

## The single-seam win

Both interactive tables (`ChessBoardView`) and the north wall (`SpectatorBoard3D`) build every
piece through **one** call — `ChessSetBuilder.BuildPiece(parent, type, white, scale, yaw)` —
and hold them as `GameObject[64] _pieces` under a `PiecesView` root. Slides, trays, captures,
and highlights all operate on those GameObjects. So a **one-line dispatch** at the top of
`BuildPiece` gives every board, every tray, and the ring's preview set 2D rendering with zero
call-site changes. This is the M8-seam discipline the repo already relies on.

## 2. Flat-sprite rendering (no custom shader)

**Geometry — `ChessSetBuilder.BuildFlatPiece(parent, type, white, scale, yaw)`.** Same recipe
as the existing `BuildLathe` path, but a single 2-triangle **quad** in the XY plane, normal
`+Z`, lying flat so it reads top-down. 12 shared `Model`s cached in a `Dictionary` (type ×
color); each quad's UVs are **baked to that piece's cell** in the atlas (§3), so one atlas +
one material serves all pieces with no per-quad attribute. Quad footprint ≈ one square
(reuse the 3D piece's local space so slides/trays inherit placement for free); offset a hair
above `BoardSurfaceZ` to avoid z-fighting the cell top. Keep the `yaw` argument working
(consider forcing `yaw:0` for flat pieces so both colours read upright from every viewing
angle — a one-line choice in `BuildFlatPiece`).

**Dispatch — `ChessSetBuilder.BuildPiece` (~535):** add `public static bool FlatMode;` and a
one-line guard at the top: `if (FlatMode) return BuildFlatPiece(...);`.

**Material — avoid a hand-written `.shader`.** The quad needs an unlit, alpha-blended,
two-sided material carrying the atlas. Ship approach, most-robust first:
- **(primary) A tiny in-repo `.vmat` art asset** `client/Assets/materials/chess_glyph.vmat`
  referencing s&box's **stock** translucent + two-sided + unlit/emissive shader, exposing the
  atlas as its colour map; `Material.Load(...)` at runtime. This is an art asset, not a
  hand-authored `.shader`, so it honours "the shader is not the play" (that rule targets the
  baked slab, which we've replaced with real per-piece quads). Alpha/two-sided/unlit flags
  live in the material, not guessed C# attribute names.
- **(fallback) Alpha-clip opaque** — if translucency proves fiddly, an opaque material with
  `alpha < 0.5` clip: order-independent, cheapest, edges slightly aliased but a top-down
  glyph reads fine.

Precedent for every API used: `FloorCheckerboard` does `Texture.Load(FileSystem.Mounted,
"textures/...")`, `Material.*`, and `renderer.SceneObject.Attributes.Set(...)`;
`ChessSetBuilder` already builds runtime meshes with `Mesh`/`Model.Builder`. **`default.vmat`
will NOT work** — it is opaque/lit and ignores PNG alpha; a dedicated translucent material is
required.

**2D board squares (white/brown).** Cells are already flat tinted `ModelRenderer`s
(`ChessRing.LightSquare`/`DarkSquare`). In the 2D respawn pass (§4) retint `_cells` to a
classic cream/brown pair; restore the 3D palette when leaving 2D. Leave 3D palettes untouched.

## 3. The 12-piece 2D art (`scripts/gen_glyph_atlas.py`)

Add a sibling generator (or `--2d` branch) writing **`client/Assets/textures/chess_glyphs_2d.png`**:
- **Layout:** 6×2 grid (`CELL=256`), row 0 = white K Q R B N P, row 1 = black; UV rect per
  cell `(col/6,row/2)…((col+1)/6,(row+1)/2)`. Quad cell = `type` index, row = `white?0:1`.
- **Outline via PIL** `stroke_width=int(CELL*0.06)`, `stroke_fill`: white piece fill
  `(245,245,245)` / stroke `(20,20,20)`; black piece fill `(20,20,20)` / stroke `(235,235,235)`.
  Per-colour outline is what makes both colours read on both white and brown squares.
- Full **RGBA** (not alpha-only like the existing atlas — that one is a shader-tinted
  silhouette; this needs real fill+outline colour). Still our own DejaVu render → CC0-clean,
  matching `Assets/ATTRIBUTION.md`. Document the re-run command in the script header.

## 4. Mode-change respawn (both views)

Both `ChessBoardView.SyncPieces` (~319) and `SpectatorBoard3D.SyncPieces` (~182) short-circuit
when the FEN string reference is unchanged — a mode change doesn't change the FEN, so add a
`int _appliedPlayModeVersion` guard at the top of each: if it differs from
`SettingsModel.SettingsVersion`, do a **full respawn** — destroy every `_pieces[i]` (+ tray
pieces / clear `_slides`,`_captured`,`_trayWhite/Black` in `ChessBoardView`), reset
`_rendered[i]='\0'`, `_lastFen=null`, retint cells for the mode, then fall through so the
existing diff respawns all 64 via the mode-appropriate builder. Factor the reset already in
`ChessBoardView.EnsureBoard` (~284) / `SpectatorBoard3D` into a shared `ResetPieces()` and call
it from both the initial bind and the mode-change path.

## 5. Top-down (nadir) camera

**Use a separate anchor — never change `SeatPitch`** (it also feeds `SeatSpotX`, i.e.
chair/walk-up placement).

- **`client/Code/World/ChessRing.cs`** — add `BuildTopAnchor(station, side)` (~near
  `BuildSeatAnchor` 1694) and assign a per-seat `WhiteTopAnchor`/`BlackTopAnchor` (~704).
  Position: directly above board center, `center + Vector3.Up * TopDownHeight` (new `[Property]
  float TopDownHeight` ≈ `SeatOrbitRadius`). Rotation: straight down (`Rotation.LookAt(
  Vector3.Down, forward)`) with `forward` toward the far rank for that seat, so each player
  reads the board from their own side. No `SeatLookDownAngle`, no `SeatEyeBlend`.
- **`client/Code/World/ChessStation.cs`** — add `WhiteTopAnchor`/`BlackTopAnchor` `[Property]`s
  + `TopAnchor(ChessSeat)` mirroring `SeatAnchor(seat)` (~32-59). Leave `SeatWorldPosition`
  (walk-up geometry) reading the orbit anchor so proximity/seating is unchanged.
- **`client/Code/World/LobbyPlayer.cs`** — `UpdateLockedCamera()` (~2399): pick the anchor by
  mode:
  ```csharp
  bool nadir = (PlayerData.Load()?.PlayMode ?? "3d-arms") == "2d";
  var anchor = nadir ? station.TopAnchor(seat) : station.SeatAnchor(seat);
  ```
  The existing lerp/slerp handles the blend; standing up already blends back via
  `_leaving`/`RestoreController()`; switching mode while seated re-reads next frame and eases
  to the other anchor for free — no extra code.

## 6. Hands & bodies off in 2D

- **Hands** — handled by `SeatedHandSpikes.HandsOn = (mode=="3d-arms")` in §1. Both `2d` and
  `3d-clean` leave hands off; pieces still slide (slide is independent of hand animation).
- **Bodies** — under a nadir camera your own seated avatar's back/arms are noise. Do **not**
  overwrite `ChessRing.TerrySeated` (an authored `[Property]`). Add a runtime override static
  **`SeatedTerry.ForceHidden`** (same static-gate shape as `HandsOn`) that the body-enable site
  ANDs against (`effectiveOn = TerrySeated && !ForceHidden`), set true only in `2d`. This keeps
  the three-deep kill-switch discipline intact and console-settable.

## AS BUILT — deviations from the plan above (2026-07-20)

Implemented on `m16-play-mode-2d`. Where the build diverged from the plan, and why:

1. **No shader, no material, no atlas — the engine's built-in `SpriteRenderer`.** Each flat piece
   is a `SpriteRenderer` fed a runtime `Sprite.FromTexture(...)` of a per-piece PNG
   (`textures/chess2d_{w|b}_{type}.png`, 12 files from `gen_glyph_atlas.py --2d`). This replaced an
   earlier custom-shader-material approach that rendered as the pink/black error material: a
   newly-added `.shader` file isn't reliably compiled/mounted in the editor (whereas the pre-existing
   `floor_checker.shader` is), so `Material.FromShader` handed back the error material. `SpriteRenderer`
   uses the engine's own pre-compiled sprite shader — always present, nothing to author. (`floor_checker`
   is the exception that proves it: it works only because it was compiled into the project back at M6.)
2. **Sprite settings do the work the shader did**: `Lighting = false` (unlit — the PNG is the final
   colour), `Opaque = true` + `AlphaCutoff = 0.5` (alpha-CLIP, order-independent, no blend/sort), and
   `Billboard = Always` — under the top-down seat camera the sprite lies flat; at other tables / the
   wall it faces the viewer as a readable card instead of going edge-on. One `SpriteRenderer` per piece
   GameObject, so slides/trays/captures/highlights that move the GameObject are unchanged.
3. **Respawn is keyed on `ChessSetBuilder.FlatMode` (a bool), not `SettingsModel.SettingsVersion`.**
   Reusing the version int would respawn every board on *any* settings change — including every
   frame of a brightness-slider drag. The bool flips only when the render-relevant mode (flat vs
   not) actually changes. `ResetPieces()` is a NEW method in each view (destroys pieces + trays,
   clears slides/`_lastFen`/`_everSynced`); `EnsureBoard` was left as-is rather than refactored, to
   avoid touching the initial-bind path.
4. **Table cell retint rides `PaintHighlights`** (mode-aware base tint + `FlatMode` in the paint
   hash so a switch forces one repaint); the wall retints explicitly via `RetintCells()`/`BaseTint()`
   since it only ever tints the two last-move squares.
5. **Anchor fields are `WhiteAnchor`/`BlackAnchor`** (not `…SeatAnchor`); the nadir pair is
   `WhiteTopAnchor`/`BlackTopAnchor` + `ChessStation.TopAnchor(seat)` (falls back to the orbit
   anchor if unbuilt). `BuildTopAnchor` looks straight down via `Rotation.LookAt(Vector3.Down,
   farDir)` — SeatPitch is untouched, as required.
6. **Body suppression** ANDs `SeatedTerry.ForceHidden` into the existing `HideLocalAvatar` choke
   point (→ `HideEveryRenderer`). A live 2D→3D switch while seated is handled: `HideLocalAvatar`
   now calls a shared `ShowHiddenRenderers()` before trimming, so the body comes back without
   standing up (factored out of `Disengage`'s undo).

**Per-machine editor step:** the real `.sbproj` is regenerated by the editor, so add
`textures/*.png` to Project Settings → Resources on each dev machine (the repo template now globs
it) or the 2D atlas 404s on joiners — same rule as the floor atlas.
